# Codex Task 10 — Documentation: mkdocs site + per-workflow tutorials

## Workflow context

Touches **all eight workflows at the documentation layer only**. By this point each workflow A–H has a tutorial markdown file in `docs/tutorials/`. This task wires up `mkdocs-material`, generates the CLI reference page from the `--help` output, ensures all tutorials run end-to-end against `data/sample/`, and adds the methodology page in its final form.

This is documentation-only. Do not change library code unless a docstring fix is required.

## Pre-task

1. Read `AGENTS.md`. Note: `mkdocs-material` and `mkdocstrings[python]` are already declared in `pyproject.toml`'s `docs` extra.
2. Read every existing `docs/tutorials/*.md`.
3. Run `pytest -q` and `pip install -e ".[docs]"`.

## What to do

1. Create `mkdocs.yml` at the repo root with:
   * `site_name: meaorganoid`
   * `theme: material` with the `slate` palette toggle.
   * Navigation:
     - Home (`README.md` rendered)
     - Quickstart (`docs/tutorials/quickstart.md`)
     - Workflows (one entry per A–H, each linking to its tutorial)
     - Methodology (`docs/methodology.md`)
     - CLI Reference (`docs/cli.md`)
     - API Reference (auto-generated by `mkdocstrings`)
     - Changelog
     - Citation
   * Plugins: `search`, `mkdocstrings` configured with `python` handler pointing at `src/meaorganoid`.
   * Markdown extensions: `admonition`, `pymdownx.details`, `pymdownx.highlight`, `pymdownx.superfences`, `tables`, `toc` with `permalink: true`.

2. `docs/cli.md` — generated content. For each subcommand, paste a fenced code block of the live `--help` output. Add a small Python script `scripts/dev/regen_cli_docs.py` that, when run, regenerates this page by calling `meaorganoid <subcommand> --help` for every subcommand and writing the result. Run it once now; commit the regenerated page.

3. `docs/api.md` — single page that uses `mkdocstrings` directives:

   ```markdown
   ::: meaorganoid.io
   ::: meaorganoid.metrics
   ::: meaorganoid.bursts
   ::: meaorganoid.compare
   ::: meaorganoid.qc
   ::: meaorganoid.plot.raster
   ::: meaorganoid.plot.spatial
   ::: meaorganoid.plot.condition
   ::: meaorganoid.connectivity
   ```

4. Confirm every public function has a NumPy docstring with `Parameters`, `Returns`, and `Examples`. Where a placeholder docstring was left in earlier tasks, fill it in.

5. Per-workflow tutorial pages: each must run end-to-end against `data/sample/` in under one minute. Each tutorial includes:
   * One-paragraph problem statement.
   * Inputs and outputs (filenames, schemas).
   * Two or three code blocks: data load, analysis call, plot.
   * The resulting figure embedded.
   * A "Public API" callout box listing the output filenames the user should depend on.

6. `docs/methodology.md` final form: one section per workflow with definitions, formulas, default thresholds, citations. The QC flag table from Task 03 lives here. The STTC section from Task 09 lives here. Burst-detection formulas from Task 04 live here.

7. Add `CHANGELOG.md` following Keep-a-Changelog format. Add an `## [Unreleased]` section. Backfill bullet-point entries for tasks 01–10 by reading the merged PRs (titles + workflow tag).

8. Add `CITATION.cff` stub at the repo root with placeholder DOI and an explicit "preprint forthcoming" note. Set `preferred-citation.type: software` for now; the user will update to `article` when the manuscript is on bioRxiv.

9. Add `.github/workflows/docs.yml` that, on push to `main`, runs `mkdocs build --strict` and uploads the site as an artifact. (Deployment to GitHub Pages can come in a later task — this just verifies the build is reproducible.)

## Tests to add

`tests/integration/test_docs_build.py` (`@pytest.mark.integration`):

* Call `mkdocs build --strict --site-dir <tmp>` via `subprocess.run`.
* Assert exit code 0 and that `<tmp>/index.html` exists.

`tests/unit/test_docstrings.py`:

* Import every module listed under "Repository layout" in AGENTS.md.
* Assert every public function (no leading `_`) has a non-empty `__doc__` containing `Parameters` and `Returns` sections.

## Success criteria

* `mkdocs build --strict` passes locally.
* `pytest -q` passes including the new tests.
* Every workflow page renders and the embedded figure shows up in the built site.
* CLI reference page is byte-equivalent to a fresh run of `regen_cli_docs.py`.
* No library code changed except docstring fills.

## What NOT to do

* Do **not** add documentation extras beyond `mkdocs-material` and `mkdocstrings[python]`.
* Do **not** auto-generate per-figure tutorials from a notebook export. Tutorials are markdown, hand-curated.
* Do **not** introduce a notebook-based tutorial format. Markdown only.
* Do **not** deploy to GitHub Pages in this task. That's later.

PR titled `docs: mkdocs site + per-workflow tutorials + cli reference` with:

* The `mkdocs build --strict` output tail.
* A link to the local preview (or screenshots of: home, one workflow tutorial, the CLI reference, and the API page).
* Tail of `pytest -q`.
* The `CHANGELOG.md` `[Unreleased]` block as inline preview.
